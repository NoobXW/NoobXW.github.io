      • less script.db | grep vuln | wc -l

Smb-vuln-ms10-061.nse

  • Smb-vuln-ms10-061.nse
    • Stuxnet蠕虫利用的4个漏洞之一
      • Stuxnet：震网病毒
    • Print Spooler服务权限不当，打印机请求可在系统目录可创建文件、可执行任意代码
    • 通过LANMAN API枚举共享打印机
    • 远程共享打印机名称
    • smb-enum-shares枚举共享
      • 身份认证参数—smbuser、smbpassword
      • nmap -p445 –script=smb-enum-shares.nse –script-args=smbuser=admin,smbpassword=pass 1.1.1.1
    • Windows XP,Server 2003 SP2,Vista,Server 2008,win7
  • 影响扫描结果的因素
    • 若没有使用smb-enum-shares脚本探测共享打印机名称，只是通过ms10-061脚本默认的LANMAN API查询，若目标系统没有开启LANMAN API，脚本就会直接判断目标系统不存在这个漏洞；其他扫描器可能通过帐号密码登录到目标系统通过查询有没有打ms10-061的补丁去判断是否存在这个漏洞，若没有打这个补丁，但没有开启共享打印机，漏洞就不会被利用
  • 原理
    • ms10-061脚本先探测目标系统是否存在共享打印机，之后通过LANMAN探测目标系统共享打印机名称，直接进行攻击性的探测；若目标系统没有开放LANMAN API，探测不到共享打印机，nmap使用枚举共享的脚本smb-enum-shares.nse探测共享打印机，再利用ms10-061去探测是否可以在系统目录可创建文件、可执行任意代码
  • 使用Smb-vuln-ms10-061.nse脚本的LANMAN API没有发现共享打印机，可以使用枚举脚本

    • nmap -p445 –script=smb-enum-shares.nse 192.18.216.167

      • 如果已知目标主机帐号密码可以使用脚本参数--script-args=smbuser=admin,smbpassword=pass登录查看是否开启共享打印机；若未知帐号密码直接使用枚举脚本即可。

OPENVAS

  • Openvas
    • Nessus项目分支
    • 管理目标系统的漏洞
    • 免费开源
    • Kali默认安装，但未配置和启动
  • OpenVAS介绍
    • 是开放式漏洞评估系统，也可以说它是一个包含着相关工具的网络扫描器。其核心部件是一个服务器，包括一套网络漏洞测试程序，可以检测远程系统和应用程序中的安全问题。
  • Openvas架构
    • OpenVAS是一个客户端/服务器架构，它由几个组件组成。在服务器上（仅限于Linux），用户需要四个程序包：
      • OpenVAS-Server: 实现基本的扫描功能
      • OpenVAS-Plugins: 一套网络漏洞测试程序
      • OpenVAS-LibNASL 和OpenVAS-Libraries: 实现服务器功能所需要的组件
      • 而在客户端上（Windows或Linux均可），用户仅需要OpenVAS客户端。
  • 官网
    • https://www.openvas.org/
    • OpenVAS从10版本后改名GVM

OPENVAS架构

  • OpenVAS Manager
    • 控制scanner和其他manager的中心组件
    • 控制中心数据库，保存用户配置及扫描结果
    • 客户端使用基于XML的无状态OMP协议与其通信
    • 集中排序筛选，使客户端获得一致展现/olos.odn
  • OpenVAS Scanner
    • 具体执行Network Vulnerability Tests (NVTS)
    • NVTs 每天通过Feed 更新
    • 受 Manager 控制
  • OSP Scanner
    • 可以统一管理多个scanner
    • 将一组 scanner 作为一个对象交给manager管理
  • Greenbone Security Assistant (GSA)
    • 提供Web servise
  • OpenVAS CLI
    • omp命令行工具，可实现批处理控制manager
  • 更新更快
    • 所有找得到的资料几乎都已不同程度的过时了

配置Openvas

  • 安装GVM

    • apt-get install gvm -y

    • gvm-setup

      • 出现报错，查看一下报错信息

        • pg_lsclusters

          • GVM启动时postgresql默认使用端口5432

          • 修改端口

          mousepad /etc/postgresql/13/main/postgresql.conf # 将13的端口修改为5433
          mousepad /etc/postgresql/15/main/postgresql.conf # 将15的端口修改为5432

          • 重启pgsql服务：systemctl restart postgresql

        • 重新运行gvm-setup

    • gvm-check-setup

      • 出现报错

      • 使用chmod 666 /var/log/gvm/openvas.log修改日志文件权限，再次验证

    • 配置外部监听

  • 基本命令

    • gvm-start #启动openvas
      gvm-stop #停止openvas gvm-check-setup #升级特征库 第一次安装，不用升级。后期使用记得经常升级特征库。

  • 基本配置

    • sudo runuser -u _gvm -- gvmd --user=admin --new-password=password #修改密码
      sudo runuser -u _gvm -- gvmd --create-user=admin2 --new-password=123456 #创建新的管理员

    • 配置文件路径

      • /lib/systemd/system/greenbone-security-assistant.service

  • https://127.0.0.1:9392

OPENVAS使用

Scans

1、向导

  • Task Wizard

      • Task Wizard 任务向导（默认向导）
      • Advanced Task Wizard 高级任务向导
      • Modify Task Wizard 自定义任务向导

    • 任务向导（默认向导：使最少的用户输入自己的配置并启动基本扫描）

      • 在输入框中输入目标系统的IP地址或主机名

        点击Start Scan ：

        → 任务向导自动执行以下步骤：

        1. 在设备上创建新的扫描目标。
        2. 在设备上创建新的扫描任务。
        3. 立即启动扫描任务。
        4. 显示页面任务 .

      • 任务启动后，可以查看进度

    • 高级任务向导

      • 除了简单向导之外，该设备还提供了一个高级向导，允许使用更多配置选项。

    • 向导修改任务（其他向导可以修改现有任务）

  • 创建任务并配置

Configuration

配置一栏主要是Scan Configs需要详细了解

Scan Configs

  • 新建扫描配置

    可以选择Openvas给定的三种扫描方式

  • 编辑新建的扫描配置

    可以自定义NVTs，选择对应的NVTs进行扫描；例如：扫描windowsOS可以只选择windows类型的NVTs；扫描LinuxOS选择Linux类型的NVTs。

    • 其中Trend一栏：

      ↗表示编辑完配置文件之后自动将官方后续更新的NVT集添加到已配置完成的配置文件中；➡表示只保存当前配置的NVT集，后续官方若有新的NVT更新，则不会自动添加到已配置完成的配置文件中

**Targets **

  • 新建扫描目标

Kali-安装Nessus

  • 官网下载基于Debian的Nessus安装包

  • 基于deb的安装包不需要使用apt-get命令，使用dpkg

  • 启动Nessus

    • /bin/systemctl start nessusd.service

  • 获取激活码

    • https://zh-cn.tenable.com/products/nessus/nessus-essentials访问官网进行注册获取注册码

  • 激活

    • kali中输入下面命令

      /opt/nessus/sbin/nessuscli fetch  --challenge

    • 将得到的challenge code复制粘贴到对应网址中

      • 第一个输入框填challenge code，第二个输入框填邮箱得到的activation code

    • 得到插件和证书下载地址

      • https://plugins.nessus.org/v2/nessus.php?f=all-2.0.tar.gz&u=4bd4b4a84a094d9c6d549da2e1b3fcde&p=6e4b71afbd6f74e272fd54824dce5242

  • 将许可证和证书上传到kali，移动到Nessus目录下

  • 输入命令进行离线激活

    /opt/nessus/sbin/nessuscli fetch --register-offline ./nessus.license

    • 出现上面的提示表示激活成功

  • 更新插件包

    /opt/nessus/sbin/nessuscli update ./all-2.0.tar.gz

  • 重启nessus服务

    • service nessusd restart

  • 重新打开web页面即可

Nessus使用

Nessus主要针对于主机的扫描，对于Web应用的扫描很薄弱

新建扫描

Settings

  • 创建新策略

    • 选择Policies点击 scan templates填写扫描信息保存

    • 查看扫描状态及结果

    若想重新配置扫描信息可以点击右上角Configure

    • Notifications

      将邮箱添加到输入框中，接收扫描结果报告

      配置SMTP Server，其中需要在邮箱中开启SMTP服务，再输入授权码才能收到邮件。

  • 发现

    • 发现栏中可以配置主机发现，端口扫描，服务发现以及从域服务中收集身份信息

  • 网络唤醒(Wake-on-LAN)

    其中主机发现当中有Wake-on-LAN选项，即使目标主机关机，可以使用Wake-on-LAN选项进行将目标主机唤醒；其中需要添加List of MAC addresses文件，Nessus会通过这些MAC地址对目标主机发送网络唤醒的指令，唤醒需要设置等待时间，等待目标主机彻底启动之后接着扫描。

Credentials

Credential一栏，若是已知目标系统的帐号密码，可以在这一栏添加目标系统的帐号密码，Nessus使用帐号密码登录到目标系统进行深度性的扫描，结果会更准确
