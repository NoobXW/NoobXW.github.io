服务扫描

  • 识别开放端口上运行的应用
  • 识别目标操作系统
  • 提高攻击效率
    • Banner捕获
      • 最简单，最不可靠，Banner信息可被管理员修改或删除，迷惑扫描者
    • 服务识别
    • 操作系统识别
    • SNMP分析
      • 简单网络管理协议
      • Community strings
      • 信息查询或重新配置
    • 防火墙识别

Banner信息

  • 软件开发商

  • 软件名称

  • 服务类型

  • 版本号

    • 直接发现已知的漏洞和软点

  • 连接建立后直接获取banner信息

    • Apache，Tomcat，版本号等

  • 另类服务识别方法

    • 特征行为和响应字段
    • 不同的响应可用于识别底层操作系统

Nc

  • nc -nv 1.1.1.1 25

    • 通过连接目标机的22端口，可以发现使用的OpenSSH软件包，Debian-ubuntu下的操作系统

  • nc -nv 1.1.1.1 80

    • 通过使用get来获取80端口banner信息

      • 得到帐号密码，后续可以猜解其他密码

Python

  • Python socket

    • Socket模块用于连接网络服务

      • 首先导入socket库，定义一个banner变量，用于TCP的连接，socket.AF_INET : IPv4 ; socket.SOCK_STREAM : 流式socket , for TCP，connect()方法用于连接目标主机，recv()方法接收4096B的数据，得到banner信息vsftpd：版本2.3.4

      • import socket
        banner = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        banner.connect(("192.168.216.167",21))
        banner.recv(4096)
        banner.close()
        exit()

    • 有些Banner信息不允许抓取，recv()函数无返回将挂起

Dmitry

  • dmitry -pb 1.1.1.1

    • -p进行TCP扫描，-b进行端口banner信息的扫描

Nmap

  • nmap -sT 1.1.1.1 -p1-100 –script=banner

    • nmap是通过脚本来获取banner信息；banner信息只能通过完整连接的方式进行获取，使用参数-sT，nmap脚本使用的语言是lua

Amap

amap专门用来发现开放端口后面跑什么服务信息，其中-B参数专门用来发现端口上的banner信息

  • amap -B 1.1.1.1 1-100

    • 可指定端范围也可单个端口扫描

服务识别

  • Banner信息抓取能力有限
  • namp响应特征分析识别服务
    • 发送系列复杂的探测
    • 依据响应特征signature
  • nc -nv 1.1.1.1 80
  • nmap 1.1.1.1 -p 80 -sV

Nmap

  • nmap通过特征来识别目标端口跑的到底是什么服务

    • nmap 192.168.216.167 -p1-100 -sV

    • -sV 用来识别端口后面跑具体是什么服务，不单单会扫描到banner信息

      • 53端口之前的工具都没有识别到具体的banner信息，通过nmap -sV扫描出是domain服务，并且http服务是由Apache和Ubuntu搭建，此时扫描的结果准确度和真实性要比banner信息高

Amap

基于特征信息的扫描

  • amap 192.168.216.167 1-100

    • 扫描结果很乱，加参数-q，发现协议信息

    • amap 192.168.216.167 1-100 -q

    • amap 192.168.216.167 1-100 -qb # -b参数打印banner信息

      • amap相比于nmap，扫描的结果还是基于banner信息，具体信息不够全面

操作系统识别

  • TTL起始值
    • Windows：128（65—128）
    • Linux/Unix：64（1—64）
    • 某些Unix：255

通过TTL值判断操作系统类型；

TTL值可被修改，迷惑扫描者。

Python

  • 使用脚本利用TTL值判断操作系统

    #!/usr/bin/python
from scapy.all import*
import logging
logging.getLogger( "scapy.runtime" ).setLevel(logging.ERROR)
import sys

if len( sys.argv ) !=2:
   print("Usage - ./ttl_os.py [IP adress]")
   print("Example - ./ttl_os.py 1.1.1.1")
   print("Example will preform ttl analysis to attemptto determine whether the systems is Windows or Linux/Unix")
   sys.exit()

ip = sys.argv[1]

ans = sr1(IP(dst=str(ip))/ICMP(),timeout=1,verbose=0)

if ans == None:
       print("No response was returned")
elif int(ans[IP].ttl)<=64:
       print("Host is Linux/Unix")
else:
       print("Host is Windows")

SNMP扫描

  • snmp
    • 信息的金矿
      • 可以查看到设备非常详细的内容
    • 经常被错误配置
    • public/private/manager
      • community：登录证书，容易被管理员遗忘修改其特征字符
      • 可用字典爆破
  • MIB Tree
    • SNMP Managerment Information Base（MIB）
      • 有些厂商有自己的MIB库
    • 树形的网络设备管理功能数据库
    • 1.3.6.1.4.1.77.1.2.25
  • onesixtyone 1.1.1.1 public
  • onesixtyone -c dict.txt -i hosts -o my.log -w 100
    • -i参数表示添加主机列表，单个主机不需要加-i参数
  • SNMP基于明文传输，也可使用网络嗅探工具进行嗅探

Onesixtyone

  • onesixtyone 1.1.1.1 public

  • onesixtyone -c /usr/share/doc/onesixtyone/dict.txt 192.168.216.157 -o my.log -w 100

    • 使用字典扫描出目标系统的community为public

Snmpwalk

  • snmpwalk 192.168.216.157 -c public -v 2c

    • -c参数指定community，-v参数指定SNMP的版本，（SNMP现在有三个版本）

      • 前面的数字表示MIB编号，可用于查询MIB库文件；查询出：主机名，MAC地址，IP地址，盘符及序列号，系统进程，系统目录等；（其中IP地址也可查询到与目标主机通信的其他主机IP）

  • nmpwalk -c public -v 2c 192.168.216.157 1.3.6.1.4.1.77.1.2.25

    • 查询某个具体的OID信息（OID：1.3.6.1.4.1.77.1.2.25）

      • 查询出系统的账号信息

Snmp-check

-t参数用于探测SNMP

  • snmp-check 1.1.1.1

    • 相对于snmpwalk，snmp-check的信息更加明确，同样可以检测出：主机名，MAC地址，IP地址，盘符及序列号，系统进程，系统目录等；（其中IP地址也可查询到与目标主机通信的其他主机IP

  • snmp-check 1.1.1.1 -w

    • -w参数检测可写权限

SMB扫描

  • Server Message Block 协议

    • 微软历史上出现安全问题最多的协议

    • 实现复杂

    • 默认开放139，445端口

    • 文件共享

      • 通过SMB协议实现

    • 空会话未身份验证访问（SMB1.0版本）

      • 不需要身份验证就可以和目标系统建立空连接

        • 可获取以下信息

        • 密码策略

        • ‘用户名

        • 组名

        • 机器名

        • 用户、组SID

SMP扫描是针对一个目标机器发现其漏洞。

Nmap

  • nmap -v -p139,445 192.168.216.0/24

    • -v参数表示显示详细结果和内容

      • 扫描192.168.216.0/24这个网段可以扫描出存活主机开放的139和445端口

    • --open参数只查看端口开放的主机

      • linux主机安装Samba等服务也会开启139和445端口，单纯扫描出来的结果无法判断是windows还是linux

  • nmap 192.168.216.157 -p139,445 –script=smb-os-discovery.nse

    • 使用nmap脚本扫描139和445端口以及发现OS信息

  • nmap -v -p139,445 –script=smb-vuln-*.nse –script-args=unsafe=1 192.168.216.157

    • 使用nmap脚本识别开放的端口上是否打了所有的补丁，是否存在漏洞，是否存在未打的补丁；smb-check-vulns脚本检查已知的SMB相关的漏洞，为此脚本中的unsafe参数指定为1：unsafe=1是整个参数赋值给--script-args；unsafe=1是对系统造成破坏的扫描；safe=1是对系统进行非破坏性的扫描。

    • 由于从NMAP 6.49beta6开始，smb-check-vulns.nse脚本被取消了。它被分为smb-vuln-conficker、•smb-vuln-cve2009-3103、smb-vuln-ms06-025、smb-vuln-ms07-029、smb-vuln-regsvc-dos、smb-vuln-ms08-067这六个脚本。用户根据需要选择对应的脚本。如果不确定执行哪一个，可以使用smb-vuln-*.nse来指定所有的脚本文件，进行全扫描。

    • smb-vuln-*.nse系列的脚本专门用于检测windows中的SMB服务漏洞，不适用于linux

      • 扫描出一个有漏洞的结果，后续可以进行验证再利用

  • nmap -v -p139,445 –script=smb-vuln-*.nse –script-args=unsafe=1 192.168.216.157 -Pn

    • -Pn参数目标机开启防火墙也可继续扫描

Enum4linux

在linux上枚举出windows系统，不支持网段扫描

  • enum4linux -a 192.168.216.157

    • 枚举出的信息很全面

      • 扫描出主机信息以及开启的服务

      • 192.168.216.157允许建立空连接

  • enum4linux -a 192.168.216.167

    • OS信息

    • 用户信息

    • 共享服务

    • 密码策略信息

SMTP扫描

针对目标系统发现邮件账号信息，后续进行社工发送钓鱼邮件等等。

  • 扫描SMTP
    • 发现邮件账号
    • 社会工程学
    • 钓鱼邮件

Nc

  • nc -nv 192.168.216.157 25 # 连接25端口
    • 使用VRFY方式尝试root用户连接

Nmap

  • nmap smtp.163.com -p25 –script=smtp-enum-users.nse –script-args=smtp-enum-users.methods={VRFY}

    • smtp-enum-users.nse枚举用户的脚本，其脚本参数smtp-enum-users.methods指定使用什么方式进行枚举，这里使用的是{VRFY}方式，默认用root账号，也可加别的参数（指定字典）

防火墙识别

  • 通过检查回包，可能识别端口是否经过防火墙过滤
  • 设备多种多样，结果存在一定 误差
  • 被过滤的端口，不是防火墙上的端口，而是内部主机向外发起请求的临时端口

第一种情况：先发送一个SYN包，对方没有回应（No），再发一个ACK包，对方回应RST，判断防火墙过滤规则为Filtered；

第二种情况：先发送一个SYN包，对方回应SYN+ACK或者SYN+RST；再发一个ACK包，对方没有回应，判断防火墙过滤规则为Filtered；

第三种情况：先发送一个SYN包，对方回应SYN+ACK或者SYN+RST；再发送一个ACK包，对方回应RST包，这种情况有可能没有过滤（Unfiltered/Open）；

第四种情况：先发送一个SYN包，对方没有回应；再发一个ACK，对方还没有回应；这种情况判断为端口是关闭状态，防火墙不允许访问，端口也没有开放。

Python

  • 使用Python脚本识别防火墙

    • #!/usr/bin/python

from scapy.all import*
import logging
logging.getLogger( "scapy.runtime" ).setLevel(logging.ERROR)
import sys

if len( sys.argv ) !=3:
   print("Usage - ./Firewalk_scan.py [Target.IP] [Target Port]")
   print("Example - ./Firewalk_scan.py 1.1.1.1 443")
   print("Example will determine if filtering exists on port 443 of Host 1.1.1.1")
   sys.exit()

ip = sys.argv[1]
port = int(sys.argv[2])

ACK_response = sr1(IP(dst=ip)/TCP(dport=port,flags="A"),timeout=1,verbose=0)
SYN_response = sr1(IP(dst=ip)/TCP(dport=port,flags="S"),timeout=1,verbose=0)

if ((ACK_response == None) or (SYN_response == None)):
   print("Port is either unstatefully filtered or host is down")
elif ((int(ACK_response[TCP].flags) == 4) or (SYN_response == None)) and not ((ACK_response == None) and (SYN_response == None)):
   print("Stateful filtering in place")   
elif int(SYN_response[TCP].flags) == 18:
   print("Port is unfiltered and open")
elif int(SYN_response[TCP].flags) == 20:
   print("Port is unfiltered and closed")
else:
   print("Unable to determine if the port is filtered")
