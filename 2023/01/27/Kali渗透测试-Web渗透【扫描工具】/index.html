Httrack

web侦察工具；对目标网站进行复制克隆，将目标网站所有的页面都复制下来到本地；目的为了减少与目标服务器进行交互，防止侦察时被发现。

  • 运行httrack
    • Enter project name :输入项目名称
    • Base path (return=/root/websites/) :拷贝下来的目标网站页面的存储位置
    • Enter URLs (separated by commas or blank spaces) :输入访问网站的起始路径

Action选项：

  1. 直接镜像，镜像站点
  2. 使用向导完成镜像
  3. 只下载某些文件（xls、doc、pdf等）
  4. 镜像当前URL的所有链接
  5. 只测试当前URL下的所有链接

  • Proxy：使用代理选项，防止IP被封

    • 免费代理网站（小心使用）：

      https://www.sabaitechnology.com/hide-my-ass/

  • You can define wildcards, like: -*.gif +www.*.com/*.zip -*img_*.zip

    Wildcards (return=none) :定义通配符，指定特定的包，例如：-*.gif +www.*.com/*.zip -*img_*.zip（*代表所有）

  • You can define additional options, such as recurse level (-r), separated by blank spaces
    To see the option list, type help
    Additional options (return=none) :定义额外的选项（-r：递归多少层查询所有的页面）

  • Ready to launch the mirror? (Y/n) :开始镜像

以上就是从dvwa网站下载到本地的dvwa目录中；下载完之后每一个页面就可以搜索信息：泄露出的内网地址，应用程序架构，email信息等

扫描工具

在Web层面的扫描工具，与通用型的扫描工具（nessus/openvas）有所不同；

基于Web的扫描工具大部分都支持两种扫描模式：1、代理截断模式；2、主动扫描模式

手动扫描：自己将所有目标页面进行探测，所有的提交表单的部分，所有的各种功能，手动登录表单，手动点击每一个页面，发现那些参数可以注入等

自动扫描：可以辅助手动扫描（手动扫描看不到一些隐藏的内容）；但也不能过于依赖自动扫描

Nikto

纯主动扫描模式

  • Perl语言开发的开源web安全扫描器
  • 软件版本

  • 避免404误判
    • 很多服务器不遵守RFC标准，对于不存在的对象返回200响应码
    • 依据响应文件内容判断，不同扩展名的文件404响应内容不同
    • 去除时间信息后的内容取MD5值
    • -no404参数：降低扫描准确度，不建议使用此参数

  • 升级数据库及插件

  • 指定端口扫描

skipfish

  • C语言编写
  • 实验性的主动web安全评估工具
  • 递归爬网
  • 基于字典的探测
  • 速度较快
    • 多路单线程，全异步网络I/O，消除内存管理和调度开销
    • 启发式自动内容识别
  • 误报较低

扫描参数说明：

-I （i）：只检查包含’string’的URL 【适用于大型网站】

-X：不检查包含’string’的URL 【如：logout】

-K：不对指定参数进行Fuzz测试

-D：跨站点爬另一个域【即目标页面中有另一个域名，则进行扫描】

-l：每秒最大请求数

-m：每IP最大并发连接数【依据带宽】

–config：指定配置文件 【将参数命令写入一个配置文件】

  • 使用skipfish必须要加-o参数

    • skipfish -o test http://192.168.216.167

    -o：后面必须跟一个目录文件，将扫描报告存入此目录

    test：指定的目录，不存在自动创建

其中扫描结果生成的报告存放在test/index.html中

  • -I指定扫描

    • skipfish -o test1 -I /dvwa/ http://192.168.216.167/dvwa/

    字符串中包含dvwa字符串的才会扫描，不包含就不扫描

  • 使用.txt文件进行批量扫描

    • skipfish -o test @url.txt

      url.txt：存放地址的文本文件

  • 指定内置字典扫描隐藏内容

    • 字典存放路径

    • skipfish -o test -S /usr/share/skipfish/dictionaries/minimal.wl http://192.168.216.167/dvwa/

    • 指定特定内容

      爬站时候发现此网站一些特有的目录内容和特征字符，可以将特征字符存下来通过-W，指定空文件a.wl再将特有的目录内容和特征字符写入其中进行扫描

      • skipfish -o test -S complet.wl -W a.wl http://1.1.1.1

  • 跨站扫描

    -D参数：当扫描当前站点时，若此目标网站有连接到新的站点的话，会直接跳到新的站点进行扫描

    • skipfish -o test3 -D www.w3.org -I /dvwa/ -S /usr/share/skipfish/dictionaries/minimal.wl http://192.168.216.167/dvwa/

  • 指定每秒最大扫描数

    • skipfish -o test3 -l 20 -I /dvwa/ -S /usr/share/skipfish/dictionaries/minimal.wl http://192.168.216.167/dvwa/

  • 每个IP的并发连接数（影响）

    • skipfish -o test3 -m 10 -I /dvwa/ -S /usr/share/skipfish/dictionaries/minimal.wl http://192.168.216.167/dvwa/

skipfish身份认证

  • 基本身份认证

    • skipfish -A user:pass -o test http://1.1.1.1

  • Cookie身份认证

    • skipfish -C “name=val” -o test http://1.1.1.1

    • 找到cookie信息

    • 构造命令

      • skipfish -o test -C "PHPSESSID=42c5a86191d270eea8ccc77eea52f9c6" -C "security=low" -I dvwa http://192.168.216.167/dvwa/

    • 按回车显示详细信息

      使用登录的方式进行扫描

  • 直接提交表单的方式将身份认证信息提交给web

    有些web使用cookie无法进行身份认证就要直接提交表单

    • man skipfish

      –auth-form ：提交表单的来源

      –auth-form-target：指定表单目标

      –auth-user-filed：指定用户名表单名

      –auth-pass-filed：指定密码表单名

      –auth-user ：提交用户名

      –auth-pass ：提交密码

      –auth-verify-url ：验证URL地址

    • skipfish -o test1 --auth-form http://192.168.216.167/dvwa/login.php --auth-form-target http://192.168.216.167/dvwa/login.php --auth-user-field username --auth-user admin --auth-pass-field password --auth-pass password --auth-verify-url http://192.168.216.167/dvwa/index.php -I /dvwa/ -X logout.php http://192.168.216.167/dvwa/

使用w3af进行NTLM身份认证扫描

NTLM微软专有的身份认证方式，相比较基本认证方式安全性很高

NTLM基于随机数，双方通过随机数各自用各自的密码进行加密，一方能够解密另一方的密码，说明双方的密码是一样的；客户端向服务器发起访问请求的时候会先提交一个随机数，服务器端会用自己的密码把随机加密，加密之后将密文回传给客户端，客户端再用自己的密码把密文解密，查看拿到的随机数与自己发送的随机是否一样，若相同则是相同的密码。在此过程中传输的是随机数，不会将明文或者密文进行传输。

基于Cookie的身份认证方式

双因素身份认证，即使输入正确表单内容也无法正常登录，需要配合Cookie进行身份认证。

  • 配置Cookie文件

cookie文件必须严格按照w3af要求的格式，必须使用Tab制表符，不能使用空格；

每一行必须存放一个cookie信息，每一个字段中间使用Tab键来分开

  1. 填写域名或者IP
  2. flag标志位：TRUE/FALSE，在这个域中所有的机器是否都可以访问这个变量，TRUE可以，FALSE不可以
  3. path：路径
  4. flag标志位：安全相关的flag，是否需要安全的连接才能够访问响应的变量
  5. 时间值：时间信息，表示cookie是否过期，这个值必须很大，值远远超过当前时间
  6. 名称
  7. 值

必须严格按照合适编辑cookie文件，否则无法使用

如果保存成功了表示cookie文件格式没有问题，若是不成功会报错

Http Sessions

手动身份认证：客户端访问目标站点后返回给客户端Session，OWASP_ZAP会自动记录由服务器端返回给客户端的Session，后续由OWASP_ZAP发起扫描请求时会自动使用记录的Session进行扫描（所谓手动就是通过浏览器手动输入用户名密码生成Session的过程）

当手动身份认证不适用的时候，可以使用表单或NTLM等其他身份认证方式

当有多个用户Session的话，OWASP_ZAP可以很方便的进行切换，不需要手动输入Session/Cookie
