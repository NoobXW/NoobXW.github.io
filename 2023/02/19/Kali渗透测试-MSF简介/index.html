Metasploit 简介

  • 基于最流行、最强大、最具扩展性的渗透测试平台软件
  • 基于Metasploit进行渗透测试和漏洞分析的流程和方法
  • 2003年由HD More发布第一版，2007年用 ruby 语言重写
    • 框架集成了渗透测试标准（PRTS）思想
    • 一定程度上统一了渗透测试和漏洞研究的工作环境
    • 新的攻击代码可以比较容易的加入框架
  • 开发活跃版本更新频繁（每周）
    • 早期版本基于社区力量维护，被 Rapid 7收购后打造出其商业版产品
    • 目前分化为四个版本，社区版本奕然十分活跃
    • HD More说：为Metasploit写书是种自虐
    • 2014年之后市场上没有再出现新的Metasploit教材

Pro 版是企业级全功能的高级渗透测试平台

MSF架构

  • Rex

    • 基础功能库，用于完成日常基础任务，无需人工手动编码实现
    • 处理 soclet 连接访问、协议应答（ҁhttp/SSL/SMB等）
    • 编码转换（XOR̵ Base64̵ Unicode）

  • Msf::Core

    • 提供 Msf 的核心基本API，是框架的核心能力实现库

  • Plugin 插件

    • 连接和调用外部扩展功能和系统

  • 技术功能模块（不是流程模块）

    • 路径：/usr/share/metasploit-framework/modules/

    • Exploits：利用系统漏洞进行攻击的动作，此模块对应每一个具体漏洞的攻击方法（主动、被动）

    • Payloads：成功exploit之后，真正的目标系统执行的代码或指令
      • Shellcode 或 系统命令
      • 三种主要的Payload：/usr/share/metasploit-framework/modules/payloads/
      • Single：all-in-one（独立载荷）
      • Stager：目标计算机内存有限时，线传输一个较小的payload用于建立连接（传输器载荷）
      • Stages：利用stager建立的连接下载的后续payload（传输体）
      • Stager、Stages都有多种类型，适用于不同场景
      • Shellcode 是payload的一种，由于其建立正向 / 反向 shell 而得名

    • Auxiliary：执行信息搜集、枚举、指纹探测、扫描等功能的辅助模块（没有paylaod的exploit模块）

    • Encoders：对payload进行加密，躲避AV检查的模块

    • Nops：提高paylaod稳定性及维持大小。在渗透攻击构造恶意数据缓冲区时，常常要在真正要执行的Shellcode之前添加一段空指令，这样当触发渗透攻击后跳转执行Shellcode时，有一个较大的安全着陆区，从而避免受到内存，地址随机化，返回地址计算偏差等原因造成的Shellcode执行失败，提高渗透攻击的可靠性。

    • Post：后期渗透模块。取得目标系统远程控制权后，进行一系列的后渗透攻击动作，如获取敏感信息、跳板攻击等操作。

MSF基本使用方法

  • 升级 / 升级

    1. apt update
    2. apt install metasploit-framework

  • msfconsole 使用接口

    • 最流行的用户接口
    • 几乎可以使用全部msf功能
    • 控制台命令支持 TAB 自动补齐
    • 支持外部命令的执行（系统命令等）

  • msf 控制台命令

    • 点击鼠标启动 / msfconsole -h -q -r -v /exit
    • help / ? / help vulns
    • banner

    • color

    在使用过程中默认使用auto模式，颜色识别

    • connect

    类似于nc，连接其他主机

    • show

    可查看各种模块内容及各个模块下的目录或文件内容

    show auxiliary / exploits / payloads / encoders / nops / post

    • search

    搜索需要查找的payload/exploit等信息

    search name:mysql / path:scada / platform:aix / type:aux /author:aaron / cve:2011 # 利用关键字多条件同时搜索

    • use exploit/windows/smb/ms08_067_netapi
      • show options / payloads / targets / advanced / evasion

    查找到相关exploit后利用use命令使用当前模块

    • info：在当前模块下使用的子命令，查看当前模块具体信息

    show options查看当前模块需要配置的信息

    show advanced，可具情况进行修改current setting/required的值

    • edit：编辑脚本或者模块内容

    • check：检查当前漏洞是否存在，向目标端口发钱连接做验证，大部分模块不支持check

    • back：在子命令提示符下返回上一级

    • db_*

    数据库默认存放位置：/usr/share/metasploit-framework/config/database.yml

    db_disconnect：断开当前数据库的连接

    db_status：查看数据库状态

    若数据未开启，开启数据库后再启动msf

    db_connect：连接其他数据库，可连接自定义数据库

    db_nmap：调用nmap，扫描结果自动保存到msf数据库中

    db_import /root/nmap.xml：导入文件

    db_export -f xml /root/bak.xml：导出扫描结果，支持xml和pwdump格式

    • hosts：查看存储到数据库中的扫描结果

    hosts <目标IP>：筛选目标主机

    hosts -u：筛选存活主机

    hosts -c address,os_flavor -S Linux：只显示address和os_flavor两列；-S参数 进行检索

    • service

    -a：添加筛选记录

    -p：筛选开放端口的主机

    -c：指定显示某些列

    -s：指定服务名称

    -u：筛选存活主机

    -S：指定字符串过滤筛选结果

    • vulns # 查看扫描出的漏洞信息
    • creds # 查看扫描出的密码信息
    • loot # 查看获取到hash值
    • set / unset # 模块下的子命令，设置/取消 目标IP等信息
    • setg /unsetg # 全局设置 / 取消全局设置

    凡是模块需要用到PHOSTS变量时候，自动就会设置为192.168.216.167

    使用setg命令后，若不是用save保存，只会在当前工作状态下有效，重新进入无效

    • save # 保存全局变量

    /root/.msf6/config存放保存配置的配置文件

    • run / exploit # 两个命令相同，执行当前模块
    • jobs # exploit执行之后在后台执行

    通常用于exploit命令的参数使用：exploit -j

    • session # 显示获得shell的会话列表

    -i：表示进入获得的shell（Shell、Meterpreter session、VNC三种shell）

    -l：列出获得shell列表

    • load / unlod / loadpath # 加载 / 卸载 外部插件

    -l：列出可调用的插件

    loadpath：加载指定目录存放的模块，比如自己开发的模块

    • route # 指定 session 路由流量

    route add 10.1.1.0 255.255.255.0 凡是法网10.1.1.0的流量都通过sid的session发到内网中

    • irb # msf开发接口

    • resource # 资源

    指定资源文件，资源文件存放配置信息

    .rc为资源文件扩展名

    可使用：msfconsole -r r.rc启动msf时候就会调用r.rc这个资源文件

Passive Exploits(被动)

针对客户端程序的漏洞。服务端放置了漏洞利用代码，客户端通过有漏洞的客户端程序访问存在了exploit代码的服务器端程序，也会导致服务端程序将客户端请求的响应报文exploit代码的内容返回给客户端，造成客户端程序漏洞被利用，就会导致payload加载，获得shell

适合客户端程序的攻击，通常配合社工进行攻击

使用模块

  • use payload/windows/shell_bind_tcp

此payload不需要配置参数，默认配置中默认绑定本地4444端口，RHOST不需要去指定，这里的RHOST表示它只接受于来自于某一个远程主机来连接目标主机，不限制来源IP只要扫描到4444这个绑定端口，任意主机都可以连接

生成payload

去除坏字符

实现坏字符剔除最终是将坏字符进行了智能Encoder编码，并不是真正的去除了坏字符

generate会根据去除的字符自动选择合适的编码方式

定制paylaod

  • generate -b '\x00\xff' -f exe -e x86/shikata_ga_nai -i 5 -k -x /usr/share/windows-binaries/radmin.exe -o /root/1.exe

-e：指定编码方式

-i：重复加密

-k：执行过程中不会产生新的进程

-x：生成payload使用模板，将模板与payload绑定在一起，radmin.exe模板文件会正常执行，同时也会执行payload

-f：输出为指定格式文件

-o：输出文件名

NOP

no-operation / Next Operation（无任何操作）

EIP返回到存储NOP sled(滑动)的任意地址时将递增，最终导致shellcode执行。

在payload前面加上NOP(可大可小)，通过修改EIP寄存器中的内容指向比较长的NOP任意位置，EIP就会从当前字节跳到下一个字节，直到滑动到payload，最终导致shellcode执行。

EIP存放下一条CPU指令的地址

  • generate -P 338

Meterpreter

  • 高级、动态、可扩展的Payload
    • 基于meterpreter上下文利用更多漏洞发起攻击
    • 后渗透测试阶段一站式操作界面
  • 完全基于内存的DLL注入式 payload（不写硬盘）
    • 注入合法系统进程并建立stager
    • 基于Stager上传和预加载DLL进行扩展模块的注入（客户端API）
    • 基于stager建立的socket连接建立加密的TLS/1.0通信隧道
    • 利用TLS隧道进一步加载后续扩展模块（避免网络取证）
  • 服务端使用C语言编写
  • 客户端提供基于ruby的全特性API
