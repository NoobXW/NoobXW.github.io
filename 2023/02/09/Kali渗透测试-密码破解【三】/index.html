ARP欺骗

发送arp欺骗包

  • arpspoof -t 192.168.216.157 -r 192.168.216.2

对win7主机说中间人是网关，告诉真实网关中间人是win7，进行双向毒化

此时win7主机的MAC地址已被欺骗为中间人MAC，网关路由器此时的MAC也是中间人的MAC

开启中间人路由功能

进行网络嗅探

使受害机产生访问流量

中间人进行网络嗅探

受害机产生的一切流量都会被中间人嗅探到，其中可以抓到受害机的用户名及密码，也可以抓取cookie伪造身份

网络嗅探工具

driftnet

将抓下来的数据包中传输的图片都还原出来

  • driftnet -i eth0

-i：设置要监听的接口，即时看到捕获的图片，图片不保存

靶机需要通过http方式来浏览图片，若是使用https方式则因数据加密而无法捕获图片。

  • driftnet -i eth0 -a -d tempdir -s

-a：不在屏幕上显示图片

-d：指定一个目录

-s：抓取声音文件，必须是MPEG格式

webspy

抓取url，不是很稳定

host参数：填被骗的客户端主机IP

  • webspy -i eth0 192.168.216.157

程序本身不稳定导致

urlsnarf

抓取url，比较稳定

  • urlsnarf -i eth0

dsniff

专门用来嗅探密码

支持解码的文件：/usr/share/dsniff/dsniff.services

  • dsniff -i eth0 -m

-m：自动进行传输协议的解码

凡是在文件中支持的协议都可以进行密码的嗅探

DNS欺骗—dnsspoof

启动DNS欺骗，让被骗的受害机访问的www.abc.com对应的`IP1`直接欺骗到`IP2`，在`IP2`上可以放置虚假的www.abc.com的登录页面达到欺骗的目的

添加虚假域名的文件：/usr/share/dsniff/dnsspoof.hosts

添加虚假域名

  • 在/usr/share/dsniff/dnsspoof.hosts文件中添加DNS欺骗的域名

IP和域名之间需要使用Tab键，不是空格

开始欺骗

  • dnsspoof -i eth0 -f /usr/share/dsniff/dnsspoof.hosts

-f：表示调用文件中设置好的域名进行欺骗

启动之后，其中并不是侦听本地53端口，而是侦听本地53端口的流量，根据受害机请求的数据包中的内容来进行内容的篡改；有哪些客户端访问DNS服务器产生的流量会被侦听，一旦发现其中请求的是*.google.com / *.taobao.com /*.youtube.com就会把回包中的内容篡改为虚假IP地址

访问虚假域名

当受害机访问某些站点就会重定向到虚假IP站点，其中在虚假IP放相应的页面，可欺骗用户名和密码

DNS欺骗代理—dnschef

流程：

攻击机通过某些手段或者漏洞修改了受害机本地DNS服务的配置，使其将DNS服务的配置指向攻击机，接着在攻击机上启动dnschef伪造几条域名解析的记录，或者解析到另外一台自己的服务器；当被修改了DNS配置之后的受害机发送DNS请求后，会将DNS请求发送给攻击机，攻击机收到请求后会返回一个错误的IP和对应关系，同时攻击机也会向真正的DNS服务器发起查询请求，真正的DNS服务器返回给攻击机正确的解析关系。

攻击前提：

需要有能力手动修改客户端网络参数的配置：

1、修改客户端hosts文件

2、修改DNS网络参数配置，指向攻击机

伪造DNS服务

  • dnschef --fakeip=192.168.216.171 --fakedomains=www.google.com,www.youtube.com --interface 192.168.216.171 -q

–fakeip：欺骗的域名要解析到那个IP上

–fakedomains：欺骗的域名

–interface：攻击机网卡对应的IP

此时已经开始监听53端口

修改受害机的网络配置

当ping测试www.google.com时，解析到的IP为192.168.216.167

当受害机查询的域名不在攻击机的域名记录中存在时，攻击机会向上游（真实域名）查询请求之后发送给客户端；当流量被重定向后，所有的流量都会经过攻击机。

中间人攻击

Ettercap

  • 统一的中间人攻击工具
  • 转发MAC与本机相同，但IP与本机不同的数据包
  • 支持SSH1、SSL中间人攻击
  • 模块划分
    • Snifer # 嗅探
      • 负责数据包转发
      • Unified
        1. 单网卡情况下独立完成三层包转发
        2. 始终禁用内核IP_Forward功能
      • Bridge
        1. 双网卡情况下的一层MITM模式
        2. 可作为IPS过滤数据包
        3. 不可在网关上使用（透明网桥）
    • MITM # 中间人
      • 把流量重定向到ettercap主机上
      • 可以使用其他工具实现MITM，ettercap之作嗅探和过滤使用
      • Bridge模式不支持SSL MITM
      • 实现MITM的方法
        • ARP
        • ICMP：ICMP路由重定向，半双工
        • DHCP：修改网关地址，半双工
        • Switch Port Stealing # 交换机端口窃取
          1. flood目标地址是本机，源地址是受害者的包
          2. 适用于ARP静态绑定的环境
        • NDP：IPv6协议欺骗技术
    • Filter # 过滤
    • Log
    • Plugin

用户界面

-T 文本界面

-G 图形界面

-C 基于文本的图形界面

-D 后台模式

指定目标

IPv4：MAC/IPs/IPv6/Ports

IPv6：MAC/IPs/IPv6/Ports

例：mac1/10.0.0.1−5,11//25,80 mac2/10.0.1.33//20−25,80,110

第二个/和第三个/之间填写IPv6，不写也需要占位

权限

需要root权限打开链路层Socket连接，然后使用nobody账号运行

日志写入目录需要nobody有写入权

ettercap主配置文件：/etc/ettercap/etter.conf

修改etter.conf：EC_UID= 0 # 赋予ettercap为root权限，能够写入日志

字符模式

  • ettercap -i eth0 -T -M arp:remote -q /192.168.1.1// /192.168.1.2// -F 1.ef -P autoadd -w a.cap -l loginfo -L logall -m messag

-T：指定字符模式

-M：指定中间人攻击方式

-q：安静模式，不会将所有流量记录下来，只记录重要信息

-F：调用过滤器

-P：添加模块

-w：抓取流量存储为.cap文件

-l：记录简单日志信息

-L：记录所有日志信息

-m：指定一个文件存储发现用户名和密码

arp:remote/single：使用全双工/单工

图形化模式

Filter

filter文件路径：/usr/share/ettercap/

etter.filter.ssh

ettercap无法对ssh2中间人作解密，只能对ssh1中间人作解密

客户端若要建立ssh2连接，中间人使用etter.filter.ssh脚本将ssh1.99强制替换为ssh1.51，这样就使客户端使用了可解密的ssh，中间人就可以解密出流经的流量；若客户端只能使用ssh2连接，则无法实现中间人攻击

编译

再使用此脚本之前需要进行编译

  • etterfilter etter.filter.ssh -o ssh.ef

  • ettercap -i eth0 -T -M arp:remote -q /192.168.216.2// /192.168.216.157// -w a.cap -l loginfo -L logall -m messag -F ssh.ef

-F：使用编译后的ssh.ef

ICMP中间人

  • -M icmp:00:11:22:33:44:55/10.0.0.1（真实网关的MAC/IP）

原理：

ICMP路由重定向实现中间人攻击；

客户端上网流量通过正常网关转发，但如果中间人向网络中发送ICMP路由重定向通知信息，告诉网络中的所有主机中间人才是最优的网络线路，在这个网络中所有客户端都接受到这个信息之后，客户端就会向中间人发送所有的流量，中间人再将包转发给网关进行转发，但由于ICMP重定向包对网关设备失效，因此回包的时候，网关不会经过中间人而是直接转发给客户端，半双工模式。

ICMP路由重定向的作用就是向网络中通告最好的路由线路，但ICMP重定向包对所有的网关设备不生效

DHCP中间人

  • −M dhcp:192.168.0.30,35,50−60/255.255.255.0/192.168.0.1（DNS）

原理：

DHCP默认情况下是核心交换机或者网关路由器作服务；

假设核心交换机提供DHCP服务，有新的主机连接到网络中，由核心交换机分配一个IP，若利用ettercap开启DHCP中间人方式，中间人会将自己也伪装成一个DHCP服务器，而且中间人会以最快的速度、在真实的DHCP服务发送响应包之前向客户端发送虚假的DHCP响应包，从而实现修改客户端网关配置，让客户端不再指向路由器网卡 ，而是指向中间人主机网卡，中间人主机再次转发给真实网关，而回包时候和ICMP相同，真实的网关会直接发送给客户端，不经过中间人，同样也是半双工模式。

交换机端口窃取中间人

  • -M port /1.1.1.1/ /1.1.1.2/

原理：

使用交换机端口窃取方式实现中间人攻击；

客户端和中间人都连接在同一台交换机上，中间人向连接交换机的端口发送flood(泛洪)ARP包，ARP包中的目的地址是中间人的MAC地址，源地址为受害机MAC地址；而交换机学习地址的时候是根据源地址学习，当中间人泛洪源地址为受害机MAC地址后，交换机就会更新连接中间人那个网口内存中记录的内容，更新为受害机的MAC地址，若此时有一个新主机要发送给目的地址为受害机的MAC地址后，本来要转发给真实的目标机，就会发送给中间人，实现交换机端口窃取，既可以是半双工，也可以是全双工。
