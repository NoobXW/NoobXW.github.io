NCAT

  • 被称为众多NC衍生版软件中最优的选择
  • 包含在Nmap工具包中
  • 具备NC所有的功能
  • 具有代理、隧道等功能。

代理功能

服务端侦听

  • ncat -l 8080 –proxy-type http –proxy-auth user:pass

-l：ncat需要指定端口，不指定端口默认侦听31337端口

–proxy-auth：身份认证

配置浏览器代理

若防火墙没有限制某些端口，例如允许22端口出站访问，就可以在公网放任何一台Linux安装nmap（自带ncat），公网Linux启用ncat代理侦听22端口，此时使用内网ip一台主机使用ncat代理公网ip实现上网

Broker中介

Broker模式的客户端相当于连接到了一个HUB，客户端之间发送任何信息都会被hub到其他客户端。

模拟场景：AB主机不通，但AC、BC主机互通，此时若A主机发送指令给C主机，由于C主机诸行了broker模式，C主机就会将A主机的指令发送给B主机，这样A主机就能达到窃取B主机信息或者获取B主机shell等目的。

启动Broker模式

  • ncat -l 333 –broker

连接Broker主机

此时假设两台主机为A、B都连接到了服务端ncat的broker模式，A主机发送I am 127.0.0.7之后B主机可以接收到，B主机发送I am localhost之后A主机也可以接收到；利用这个特性就可以执行操作系统指令。

执行系统指令

连接的同时执行系统指令，其中执行的系统指令必须使用``符号引起来

批量执行命令：ncat 1.1.1.1 –sh-exec “echo pwd“

批量传文件： ncat –send-only 1.1.1.1 < inputfile

SOCAT

  • 被称为nc++（增强增强版的nc）
  • 双向数据流通道工具

连接端口

  • socat - tcp:1.1.1.1:80

-：表示标准的输入输出，将数据输入给socat

侦听端口

  • socat - tcp4-listen:22 / socat - tcp-l:333

接收文件

  • socat tcp4-listen:333 open:2.txt,creat,append

open:2.txt,creat,append：通过open指令creat创建2.txt，后续若有数据传输使用append添加到2.txt中

发送文件

  • cat 1.txt | socat - tcp4:1.1.1.1:333

远程shell

  • socat tcp-l:23 exec:sh,pty,stderr

exec：执行指令

exec:sh,pty,stderr：执shell，将pty终端包括stderr(标准输入输出的错误信息)传过去

服务端侦听

客户端接收

端口转发

  • socat tcp4-listen:8080,fork tcp4:1.1.1.1:22

fork：此时侦听一个端口，若有一个客户端连接此端口，会新启用一个新的子进程接收这个客户端的连接，如果又来一个客户端连接8080端口，同样也会复刻当前进程接收新客户端连接

远程执行命令

服务端侦听

  • socat - udp-l:2001

客户端连接

  • echo "id" | socat - udp4-datagram:1.1.1.1:2001

udp4-datagram：表示发送的数据段

UDP 全端口任意内容发包

  • for PORT in {1..65535}; do echo "aaaaa" | socat - UDP4-DATAGRAM:1.1.1.1:$PORT; sleep .1; done

PORT从7-65535逐一取值，在循环中执行do echo “aaaaa”，将这些数据段通过管道发送给服务器

此时就可以通过socat向目标主机的任意端口发送任意数据段的数据包，若端口开放服务器会返回一个响应包，若端口未开放，会返回一个ICMP包

二进制编辑器

假如有一个执行程序，这个执行程序编译之后都是二进制文件，若想对这些二进制文件进行修改的话，也可以使用socat；

例：

echo -e "\0\14\0\0\c" | socat -u - file:/usr/bin/squid.exe,seek,seek=0x00074420

-e：输入二进制内容

-u：单向数据传输，传给/usr/bin/squid.exe

seek：指定二进制文件中的偏移位置

简单的WEB服务器

socat -T 1 -d -d TCP-L:10081,reuseaddr,fork,crlf SYSTEM:"echo -e \"\\\"HTTP/1.0 200 OK\\\nDocumentType: text/plain\\\n\\\ndate: \$\(date\)\\\nserver:\$SOCAT_SOCKADDR:\$SOCAT_SOCKPORT\\\nclient: \$SOCAT_PEERADDR:\$SOCAT_PEERPORT\\\n\\\"\"; cat; echo -e \"\\\"\\\n\\\"\""

Ptunnel

  • Ping tunnle ICMP隧道工具
    • 通过ICMP echo（ping requests）和reply（ping reply）实现隧道
    • 适用于只允许ping出站流量的环境
    • 支持多并发连接、性能优
    • 支持身份认证
  • ptunnel 0.72版本存在拒绝服务漏洞

服务端运行

  • ptunnle -x 123456

-x：设置隧道连接密码

客户端运行

  • ptunnel -p proxy_ip -lp 2222 -da destination -dp 80 -x 123456

-p proxy_ip：指定ptunnel服务器端

-lp：客户端本地侦听的端口

-da destination：目标资源地址（最终访问的资源）

-da：目标资源端口

-x：服务器密码

建立连接之后，由于没有任何信息通信，服务器端开始时候抓不到ICMP包

产生访问流量

由于客户端侦听的是本地的2222端口，一旦访问本地的2222端口，访问的请求流量就会通过建立好的隧道发送给服务端，再有服务端转发给192.168.216.172:80

嵌套SSH隧道

ptunnel建立好隧道之后，监听本地3333端口，只要访问本地3333端口就会转发到目标服务器的22端口后建立ssh动态端口转发

Proxytunel

  • 通过标准的HTTP / HTTPS代理创建隧道的工具
  • 通过HTTP CONNECT 方法封装信息
  • 适用于内网使用代理并且防火墙只允许代理服务器上网的场景
  • 无法创建DNS隧道和ICMP隧道

场景一

内网代理安装squid3，内网PC安装proxytunnel

访问外网

建立隧道

  • proxytunnel -a 80 -p 192.168.216.181:3128 -d 192.168.216.172:80

-a：指定侦听本地端口

-p：proxy，指定内网代理服务器ip

-d：公网资源IP

本地访问127.0.0.1:80之后流量会转发给代理服务器，代理服务器通过防火墙允许的规则之后访问指定的公网资源。

其中公网端口的指定是根据防火墙规则而定的，若防火墙只允许内网代理服务器访问公网80端口，就只能访问允许的端口。

其他的http_access等字段与场景一的相同

-r：指定公网代理IP

SSLH

  • 端口分配器
    • 根据客户端第一个包检测协议类型
    • 根据协议检测结果将流量转发给不同目标
    • 支持HTTP，HTTPS，SSH，OpenVPN，tinc，XMPP和其他可基于正则表达式判断的协议类型（自己写正表达式判断其他协议类型）
    • 适用于防火墙允许443端口入站访问流量的环境
  • sslh先判断第一个数据包访问的是什么资源sslh就会转发给相应的资源。例如：假设控制了一台https主机，在此主机上安装sslh软件侦听443端口，，让原来的https服务侦听其他端口，；若有客户端访问公网的443端口，流量被防火墙映射到sslh主机的443端口，sslh先判断第一个数据包是什么协议类型，若是https请求，就进行本地转发到真正的https服务，响应包继续原路返回到外网客户端。

配置sslh

  • vim /etc/default/sslh

客户端连接

访问http服务

客户端使用http访问443端口经过防火墙映射到内网主机的sslh主机，sslh经过判断第一个包是http服务就会转发到对应的http服务上

访问ssh服务

通过ssh访问sslh内网主机的443端口，会被转发到内网主机的ssh的22端口

若使用nmap对防火墙外网口进行扫描，会扫描出443一个端口跑的三个服务：https，hhtp和ssh（一个端口提供多个服务）

Stunnel4

  • 无需修改原代码的情况下将TCP流量封装于SSL通道内
  • 适用于本身不支持加密传输的应用
  • 支持openssl安全特性
  • 跨平台
  • 性能优

内网部署Stunnel4 Server

生成证书

  • openssl req -new -days 365 -nodes -x509 -out /etc/stunnel/stunnel.pem -keyout /etc/stunnel/stunnel.pem

req -new：申请新证书

-day：证书期限

-nodes：不要加密生成的密钥

-x509：通用证书标准

-out：输出证书的路径，stunnel证书是.pem格式，包含私钥，而.cer格式不包含私钥只有公钥

-keyout：key(私钥)保存在stunnel.pem中

创建stunnel配置文件

  • vim /etc/stunnel/stunnel.conf
# /etc/stunnel/stunnel.conf

cert = /etc/stunnel/stunnel.pem  # 证书路径
setuid = stunnel4  # 设置运行服务账号为stunnel，stunnel4为stunnel的uid
setgid = stunnel4  # 设置运行服务账号为stunnel，stunnel4为stunnel的gid
pid = /var/run/stunnel4/stunnel4.pid
[mysqls]                          # 定义访问资源
accept = 0.0.0.0:443              # 本机侦听443端口
connect = 192.168.216.167:3306    # 连接目标服务器192.168.216.167的3306端口

配置Stunnel4自启动

  • vim /etc/default/stunnel4

启动stunnel4服务

  • service stunnel4 start

解决报错

在启动stunnel4服务后出现了报错

  1. 此处提示Process进程报错，查看进程信息后的确没有3368进程运行，而在上面的服务端配置文件中pid = /var/run/stunnel4/stunnel4.pid，进程号在stunnel4.pid中，但在stunnel4文件夹中没有stunnel4.pid进程文件
  2. 去掉所有注释

添加进程号
第一步：在/var/run/stunnel4/路径下执行 vim stunnel4.pid
第二步：在stunnel4.pid文件中写入3368进程号（根据给出的进程报错信息填写对应进程号）
去掉注释

将配置文件中所有注释去掉

启动服务

此时就能正常启动stunnel4服务侦听本地443端口

Stunnel4客户端配置

创建配置文件

  • vim /etc/stunnel/stunnel.conf
client = yes                   	  # 证明自己是client
[mysqls]                          # 定义与Server相同的访问资源

connect = 192.168.216.181:443     # 隧道另一端访问的Server，要符合防火墙规则

客户端自启动

启动客户端
